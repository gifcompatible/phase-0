Version Control, Git, and GitHub

Wednesday December 2nd, 2015

Version control is similar to having multiple saves in a video game instead of one save file you overwrite; it allows you to see when you saved, what you were doing when you saved, and you can go back to that save if you mess up later in the game. Version control is not just used by developers-it can be used for any number of things to keep track of overall progress on a project. The great thing about version control is that it helps prevent fatal mistakes from being made. Think of it as a series of do-overs.

Once we have completed our desired changes and updates, we can use git to commit and push our code to our GitHub repository. GitHub is a web service that allows individuals and teams to manage projects and to collaborate and contribute to one another's projects in a nondestructive way. GitHub is useful because it stores the code so that it can be accessed and edited by others using what are called "forks". Forks allow one to copy the code of another for the purpose of adding or, in some cases perhaps, removing features. GitHub maintains extensive records of every new "commit" or update, who did it and any comments or descriptions about that particular action. It is easy to develop a timeline of changes and the thinking and personalities involved. GitHub also functions as social media, where one can view the projects and contributions of other users. This is concept is playing a larger and larger role in the fiding work in the software development industry, as it is easy for potential employers to see the projects a prospective employee has completed.
